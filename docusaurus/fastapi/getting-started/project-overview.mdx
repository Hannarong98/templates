Depending on which template you chose,
your project structure will look more or less like the following:

```
├── alembic.ini                                 # Configuration for your migrations
├── app  # This is where your FastAPI lives
│     ├── api  # APIs and routes
│     │     ├── api_v1
│     │     │     ├── api.py                    # API routes
│     │     │     ├── endpoints                 # Yor API endpoints
│     │     │     │     ├── __init__.py
│     │     │     │     ├── cars.py             # Example database API
│     │     │     │     └── views.py            # Example API with authentication
│     │     │     └── __init__.py
│     │     ├── __init__.py
│     │     ├── dependencies.py                 # Dependencies for your application
│     │     └── security.py                     # Security for your application, such as Azure AD auth
│     ├── core
│     │     ├── __init__.py
│     │     ├── db.py                           # Database engine is configured here
│     │     └── config.py                       # All your application settings
│     ├── __init__.py
│     ├── main.py                               # FastAPI is configured here. This is where everything is put together.
│     ├── models                                # Database models
│     │     ├── __init__.py
│     │     └── cars.py
│     └── schemas                               # Pydantic models
│         ├── __init__.py
│         └── hello_world.py
├── ci
│     └── docker
│         └── fastapi
│             ├── Dockerfile
│             ├── entrypoint_dev.sh
│             ├── entrypoint.sh
│             └── gunicorn.conf.py
├── docker-compose.yaml
├── migrations                                  # Autogenerated migrations are put here
│     ├── __init__.py
│     ├── alembic_functions.py
│     ├── env.py
│     ├── README
│     ├── script.py.mako
│     └── versions
│         ├── __init__.py
│         └── d8ba1a4996fa_init.py              # This is the first migration file
├── mypy.ini                                    # mypy configuration files
├── poetry.lock                                 # Autogenerated file for poetry
├── pyproject.toml                              # Your poetry configuration
├── pytest.ini                                  # Settings for all your tests
└── tests                                       # All tests are put here
    ├── __init__.py
    ├── api
    │     ├── __init__.py
    │     ├── auth_utils.py
    │     ├── conftest.py
    │     ├── test_api_with_auth.py
    │     └── test_cars_api.py
    ├── conftest.py
    ├── models
    │     ├── __init__.py
    │     └── test_cars.py
    └── README.md
```
